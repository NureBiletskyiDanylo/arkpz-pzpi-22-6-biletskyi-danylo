	Розгорнути програмну систему та продемонструвати її роботу. 
3.	Хід роботи:
3.1. Налаштування бази даних у Azure:
Для початку налаштування бази даних був створений акаунт Azure та resource group під назвою MediStoS. 
Далі був створений новий гнучкий сервер бази даних (Azure Database for PostgreSQL - Flexible Server). 
Після його створення та обрання налаштувань системи, таких як розмір диску, оперативної пам’яті та процесору, була створена база даних PostgreSQL під назвою medistosdb. 
Такі дані як: server name; server admin login name; password, були потім використані для налаштування з’єднання API та бази даних.

3.2. Розгортання АРІ у Azure:
Для розгортання АРІ у Azure, для початку був клонований репозиторій проекту, а саме: https://github.com/NureBiletskyiDanylo/arkpz-pzpi-22-6-biletskyi-danylo/tree/main. 
Після його клонування, проект був відкритий за допомогою Visual Studio. 
В ній був відкритий файл apisettings.json, в якому був налаштований раніше вказаний шлях до бази даних, а також дані smtp акаунту для надсилання повідомлень. 
Зокрема у Visual Studio я увійшов в акаунт Майкрософт, на якому розташовується Azure. 
Після цього я за допомогою натискання на праву кнопку миші по проекту опублікував (Publish) його в Azure у resource group MediStoS. 
Після виконання всіх операцій на самому сервісі Azure, АРІ вважався розгорнутим.

3.3. Міграція даних після розгортання:
Після розгортання продукту, для міграції даних необхідно використовувати інструмент PgAdmin 4, створений спеціально для бази даних PostgreSQL. 
За допомогою нього, ви можете напряму підключитися до серверу бази даних в Azure, та завантажити дані з .sql резервної копії чи з інших. 
Отримати .sql резервну копію можна або за допомогою цього ж інструменту PgAdmin 4, або за допомогою ендпоінту api.
